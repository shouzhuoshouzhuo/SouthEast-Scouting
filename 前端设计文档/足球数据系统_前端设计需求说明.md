# 足球数据系统（球员对比页）前端设计需求说明

> 版本：V1.0（仅前端设计阶段，不含算法实现）  
> 目标：支持五大联赛球员能力对比分析，核心为“最多 3 人对比 + 雷达图可视化”。

---

## 1. 页面目标与范围

本阶段目标是完成一个可落地的 Web 前端页面设计，用于展示球员能力对比分析结果。

- 核心能力：
  1. 用户选择球员进行对比（最多 3 名）
  2. 球员输入支持 full name 与部分名模糊匹配，自动补全候选列表
  3. 页面展示每位球员的基础信息卡
  4. 雷达图叠加展示多维指标对比
  5. 支持联赛、赛季、位置等筛选条件
- 非目标（本阶段不做）：
  - 指标算法设计与后端计算逻辑
  - 模型训练、预测能力
  - 数据采集/ETL 细节

---

## 2. 核心业务理解（已确认）

### 2.1 对比对象
- 对比主体：球员
- 最大对比数量：3 人
- 支持同位置球员对比（默认）
- 数据来源为已计算后的指标值（前端仅展示）

### 2.2 核心可视化
- 核心图表：雷达图（Spider/Radar Chart）
- 雷达图每一轴对应一个能力指标
- 每位球员对应一组颜色（线条 + 半透明填充）
- 支持图例联动（隐藏/显示某球员）

### 2.3 信息呈现
每个球员需要在页面顶部或图表周边展示独立信息卡，至少包含：
- 球员姓名
- 比较基准说明（例如 Percentile rank vs 同位置）
- 赛季与联赛信息
- 颜色标识（与雷达图保持一致）

---

## 3. 页面信息架构（IA）

建议采用“三段式结构”：

1. **筛选与选择区（顶部）**
   - 联赛选择（五大联赛）
   - 赛季选择
   - 位置选择（前锋/中场/后卫/门将）
   - 球员搜索与添加（最多 3 人）

2. **球员对比卡片区（中上）**
   - 按已选择顺序显示 1~3 张球员卡
   - 每张卡含：头像（可选）、姓名、球队、赛季联赛、对比维度说明、移除按钮

3. **图表与解释区（中下）**
   - 主体：雷达图
   - 右侧/下方：指标说明列表（指标含义、单位、是否标准化）
   - 可扩展：当前选中指标的 tooltip 解释

---

## 4. 关键交互流程

### 4.1 选择球员流程
1. 用户先选联赛/赛季/位置
2. 在搜索框输入球员 full name 或部分名
3. 实时弹出匹配候选（下拉列表），展示完整姓名（可附球队/联赛用于区分重名）
4. 支持键盘与鼠标选择（↑/↓ 切换，Enter 确认，Esc 关闭）
5. 选中候选后添加到对比区
6. 添加到 3 人后，继续添加应提示“最多选择 3 人”
7. 支持替换或移除某个已选球员

### 4.2 对比展示流程
1. 只选 1 人：展示单人雷达画像
2. 选 2 人：双人叠加对比
3. 选 3 人：三人叠加对比（与示例图一致）
4. 图例点击可临时隐藏某球员曲线

### 4.3 交互反馈
- 无数据时：展示空状态（请先选择球员）
- 数据加载中：骨架屏/加载动画
- 数据异常：友好错误提示 + 重试按钮

## 5. UI/视觉设计建议（前端实现导向）

### 5.1 风格建议
- 深色背景 + 高对比强调色（与示例视觉一致）
- 颜色规范：每个球员一个固定色，避免混淆
- 文案风格：专业、简洁、偏数据分析语气

### 5.2 雷达图设计要点
- 轴数量建议 6~12（过多会拥挤）
- 轴标签支持缩写 + hover 全称
- 数值建议统一到可对比区间（如 0~100）
- 支持 hover 查看具体值（tooltip）

### 5.3 响应式建议
- 桌面端优先（1920、1440 宽度适配）
- 窄屏下：卡片区换行，图表区域优先保证可读性

## 6. 前端组件清单（不涉及代码）

1. 筛选组件
   - 联赛下拉
   - 赛季下拉
   - 位置下拉
2. 球员选择组件
   - 搜索输入框（支持 full name / 部分名）
   - 自动补全下拉列表（候选全名 + 辅助信息）
   - 已选球员标签/卡片
3. 对比信息卡组件
   - 颜色条
   - 名称与元数据
   - 删除/替换操作
4. 雷达图组件
   - 多数据集叠加
   - 图例联动
   - Tooltip
5. 状态组件
   - 空状态
   - 加载状态
   - 错误状态

## 7. 数据契约（前端视角）

> 仅定义前端需要的字段结构，不涉及后端算法。

### 7.1 球员基础信息
- playerId
- playerName
- teamName
- league
- season
- position
- colorToken（前端映射色）

### 7.2 指标数据（已计算）
- metrics: Array
  - metricKey（如 interceptions）
  - metricLabel（如 Interceptions）
  - metricValue（如 78.2）
  - metricUnit（可选）

### 7.3 页面元信息
- compareContext（如 vs 同位置 percentile）
- updatedAt（数据更新时间）

---

## 8. V1 验收标准（建议）

1. 可完成联赛/赛季/位置筛选
2. 输入球员 full name 或部分名时，可返回匹配候选全名列表
3. 候选列表支持键盘与鼠标选择并可成功添加球员
4. 可选择 1~3 名球员进行对比
5. 雷达图可正确叠加展示三名球员指标
6. 颜色映射一致（卡片与图表一致）
7. 支持移除球员并即时刷新图表
8. 基础异常状态可用（空、加载、错误）

---

## 9. 后续扩展方向（V2+）

- 增加“联赛内排名/五大联赛排名”切换视图
- 增加指标分组（进攻/组织/防守）
- 增加球队表现页（进攻效率/有球效率/无球效率）
- 增加导出报告（PNG/PDF）
- 增加球员历史赛季趋势对比

---

## 10. 结论

当前需求可归纳为：

**做一个以雷达图为核心的球员能力对比前端页面，支持最多 3 人同屏对比，前端展示已计算指标，重点在信息结构清晰、交互流畅、视觉可读。**
